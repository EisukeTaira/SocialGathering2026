# バレーボール試合状況管理システム 詳細設計書

## 1. プロジェクト概要
本システムは、6つのバレーボールコートの試合進行状況をリアルタイムで管理・可視化するためのWebアプリケーションです。特に、エントリーチームの一覧から始まり、最終的な優勝者が決まるまでの「トーナメントの流れ」を直感的に把握できることを目指します。

## 2. 主要機能
### 2.1 リアルタイム試合状況（Overview）
- 6コート分の現在の対戦カード、スコア、審判チームを一覧表示。
- Googleスプレッドシートの更新を数秒間隔で自動反映。

### 2.2 垂直上昇型トーナメント表（Bracket）
- 下から上へ勝ち上がっていく「ピラミッド型」の樹形図スタイル。
- **勝者自動反映**: スプレッドシートで勝者を記録すると、自動的に次ラウンドの対戦カードへ名前が移動。

### 2.3 エントリーチーム一覧（Team Entries）
- 各コートに参加する全チームをリスト表示。

### 2.4 管理・入力システム（Admin）
- 各コートの対戦チーム名、スコア、審判、試合ステータスの更新。
- モバイル端末からの操作も考慮したUI。

## 3. デザイン指針 (UI/UX)
- **カラーコンセプト**: 完全な「モノクローム（白・黒・グレー）」を採用。
- **美学**: ミニマルかつプレミアムな質感。太いボーダー、大胆なタイポグラフィ（Inter/Roboto）、微細なアニメーション。
- **視認性**: 遠くからでも、あるいはスマートフォンでも、現在の得点が瞬時に読み取れるコントラスト。
- **装飾**: 試合エリアには薄い背景色（例：淡いイエローベージュ `#fdfdf0`）をアクセントとして使用。

## 4. 技術アーキテクチャ
- **データ基盤**: Googleスプレッドシート（DBとして利用）。
- **通信層**: Google Apps Script (GAS) Web App API。
- **フロントエンド**: HTML5 / CSS3 / Vanilla JavaScript。
- **描画エンジン**: 
    - レイアウト: CSS Flexbox / Grid。
    - 接続線: SVG (Scalable Vector Graphics)。

## 5. データ構造（スプレッドシート）
- **Matchesシート**: 現在の試合状況（コート別）。
- **Bracketシート**: トーナメント構造（ID, ラウンド, 試合順, 次の試合ID等）。
- **Teamsシート**: コート別の参加チーム一覧。

## 6. 開発フェーズ
1. **設計・要件定義**: 現在の設計書（plan.md）の合意。
2. **基盤実装**: GAS APIの構築と通信テスト。
3. **UI/UX実装**: モノクロデザインの適用とレスポンシブ対応。
4. **機能実装**: トーナメント描画ロジック（SVG）とチーム一覧の統合。
5. **テスト・調整**: 実機での動作確認と表示の微調整。

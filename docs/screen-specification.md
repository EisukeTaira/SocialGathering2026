# バレー大会 Webアプリ 画面仕様書

## 1. 文書概要

### 1.1 目的

本書は、Index、Tournament、ScoreInput、Admin の各ページについて、
ワイヤーフレーム相当の画面構成、表示項目、操作、状態変化を定義するための画面仕様書である。

### 1.2 対象読者

- 実装担当者
- デザイン担当者
- 大会運営担当者

### 1.3 関連文書

- docs/basic-design.md

## 2. 共通仕様

### 2.1 共通レイアウト

- 画面上部に大会名と共通ナビゲーションを配置する。
- 主表示領域は 1 カラムを基本とし、PC では必要に応じて 2 カラム以上へ展開する。
- 画面下部に最終更新時刻を表示できる領域を持つ。

### 2.2 共通ナビゲーション

| 項目 | 内容 |
| --- | --- |
| Index | 大会全体ダッシュボードへ遷移 |
| Tournament | コート別トーナメント一覧へ遷移 |
| ScoreInput | 審判入力画面へ遷移 |
| Admin | 管理画面へ遷移 |

### 2.3 共通状態表示

| 状態 | 表示方針 |
| --- | --- |
| scheduled | 未開始のバッジを表示 |
| in_progress | 進行中の強調表示を行う |
| completed | 終了として表示 |
| locked | 修正制限中として表示 |

### 2.4 共通メッセージ

- 成功メッセージは画面上部または対象フォーム直下に表示する。
- エラーメッセージは該当項目の近くに表示する。
- 通信導入後はローディング状態を表示できる設計とする。

## 3. P01 Index ページ仕様

### 3.1 目的

大会全体の進行、固定タイムテーブル、各コートの試合状況を俯瞰できるようにする。

### 3.2 想定利用者

- 参加者
- 観客
- 運営者

### 3.3 主な表示ブロック

| ブロックID | ブロック名 | 内容 |
| --- | --- | --- |
| IDX-01 | ヘッダー | 大会名、日付、共通ナビ |
| IDX-02 | タイムスケジュール | 固定タイムテーブル一覧 |
| IDX-03 | 全体サマリー | 進行中試合数、終了試合数、入力受付状態 |
| IDX-04 | コート状況一覧 | A〜F コートの現在試合と次試合 |
| IDX-05 | お知らせ | 運営メッセージ、更新時刻 |

### 3.4 表示項目詳細

| 項目 | 表示内容 |
| --- | --- |
| 大会名 | 大会の正式名称 |
| 開催日 | 当日日付 |
| タイムテーブル | 時刻、対象コート、補足 |
| コート名 | A〜F コート |
| 現在試合 | 対戦カード、試合順、状態 |
| 次試合 | 次の対戦カード、予定時刻 |
| 入力受付状態 | 受付中または停止中 |
| 更新時刻 | 最終更新日時 |

### 3.5 ワイヤーフレーム

```text
+--------------------------------------------------------------+
| 大会名                                      [更新時刻]       |
| [Index] [Tournament] [ScoreInput] [Admin]                  |
+--------------------------------------------------------------+
| 固定タイムテーブル                                           |
| 09:00  A1  B1  C1  D1  E1  F1                               |
| 09:20  A2  B2  C2  D2  E2  F2                               |
| 09:40  A3  B3  C3  D3  E3  F3                               |
+--------------------------------------------------------------+
| 全体サマリー                                                 |
| 進行中: 3  終了: 12  受付状態: 受付中                        |
+------------------+------------------+-----------------------+
| Aコート           | Bコート           | Cコート                |
| 現在: A1 vs A2    | 現在: B1 vs B2    | 現在: C1 vs C2         |
| 状態: 進行中      | 状態: 未開始      | 状態: 終了             |
| 次:   A3 vs A4    | 次:   B3 vs B4    | 次:   C3 vs C4         |
+------------------+------------------+-----------------------+
| Dコート           | Eコート           | Fコート                |
| 現在: D1 vs D2    | 現在: E1 vs E2    | 現在: F1 vs F2         |
| 状態: 進行中      | 状態: 未開始      | 状態: 進行中           |
| 次:   D3 vs D4    | 次:   E3 vs E4    | 次:   F3 vs F4         |
+--------------------------------------------------------------+
| お知らせ: 試合開始5分前には各コートへ集合                    |
+--------------------------------------------------------------+
```

### 3.6 操作仕様

| 操作 | 挙動 |
| --- | --- |
| コートカード選択 | 該当コートの Tournament 表示へ遷移 |
| 共通ナビ選択 | 対応ページへ遷移 |
| 更新ボタン配置時 | 最新情報を再描画 |

### 3.7 表示ルール

- 途中得点は表示しない。
- 現在試合が存在しない場合は、次回試合または待機中を表示する。
- 次試合が未登録の場合は、予定未登録と表示する。

### 3.8 レスポンシブ方針

- モバイルではコートカードを 1 列で表示する。
- タイムテーブルは横スクロールを許容する。

## 4. P02 Tournament ページ仕様

### 4.1 目的

特定コートの試合順、結果、勝敗を一覧で確認できるようにする。

### 4.2 想定利用者

- 参加者
- 観客
- 運営者

### 4.3 主な表示ブロック

| ブロックID | ブロック名 | 内容 |
| --- | --- | --- |
| TRN-01 | ヘッダー | 大会名、共通ナビ |
| TRN-02 | コート切替 | A〜F コート切替タブ |
| TRN-03 | コート概要 | コート名、所属チーム数、当日試合数 |
| TRN-04 | 試合一覧 | 試合順、対戦カード、スコア、勝者、状態 |

### 4.4 表示項目詳細

| 項目 | 表示内容 |
| --- | --- |
| コート名 | 表示対象コート |
| 所属チーム数 | 固定所属チームの数 |
| 試合順 | コート内の試合番号 |
| 対戦カード | チームA vs チームB |
| スコア | 1 セット分の得点 |
| 勝者 | 勝利チーム名 |
| 状態 | 未開始、進行中、終了、修正制限中 |
| 開始予定時刻 | 固定タイムテーブルの時刻 |

### 4.5 ワイヤーフレーム

```text
+--------------------------------------------------------------+
| 大会名                                                       |
| [Index] [Tournament] [ScoreInput] [Admin]                  |
+--------------------------------------------------------------+
| [A] [B] [C] [D] [E] [F]                                     |
+--------------------------------------------------------------+
| Aコート  所属: 7チーム  試合数: 21                           |
+--------------------------------------------------------------+
| No | 時刻  | 対戦カード       | スコア | 勝者 | 状態         |
| 01 | 09:00 | A1 vs A2         | 21-18  | A1   | 終了         |
| 02 | 09:20 | A3 vs A4         | -      | -    | 未開始       |
| 03 | 09:40 | A5 vs A6         | -      | -    | 未開始       |
| 04 | 10:00 | A1 vs A3         | -      | -    | 未開始       |
+--------------------------------------------------------------+
```

### 4.6 操作仕様

| 操作 | 挙動 |
| --- | --- |
| コートタブ選択 | 対象コートの試合一覧へ切替 |
| 試合行選択 | 将来的に詳細モーダル表示へ拡張可能 |

### 4.7 表示ルール

- スコア未入力時はハイフンまたは未入力ラベルを表示する。
- 勝者は終了試合のみ表示する。
- locked 状態では修正制限中ラベルを追加表示する。

### 4.8 レスポンシブ方針

- モバイルでは表をカード形式へ変換する。
- コート切替タブは横スクロールを許容する。

## 5. P03 ScoreInput ページ仕様

### 5.1 目的

審判が担当試合の結果を短時間で正確に入力できるようにする。

### 5.2 想定利用者

- 審判

### 5.3 主な表示ブロック

| ブロックID | ブロック名 | 内容 |
| --- | --- | --- |
| SCI-01 | ヘッダー | 大会名、共通ナビ |
| SCI-02 | 入力受付状態 | 入力可否メッセージ |
| SCI-03 | 入力フォーム | コート、試合、スコア入力 |
| SCI-04 | 完了メッセージ | 入力完了またはエラー表示 |

### 5.4 入力項目仕様

| 項目ID | 項目名 | 形式 | 必須 | 備考 |
| --- | --- | --- | --- | --- |
| SCI-F01 | コート名 | セレクト | 必須 | A〜F |
| SCI-F02 | 対象試合 | セレクト | 必須 | コート選択後に絞り込み |
| SCI-F03 | チームA名 | テキスト表示 | 自動 | 編集不可 |
| SCI-F04 | チームB名 | テキスト表示 | 自動 | 編集不可 |
| SCI-F05 | チームAスコア | 数値入力 | 必須 | 0以上 |
| SCI-F06 | チームBスコア | 数値入力 | 必須 | 0以上 |

### 5.5 ワイヤーフレーム

```text
+--------------------------------------------------------------+
| 大会名                                                       |
| [Index] [Tournament] [ScoreInput] [Admin]                  |
+--------------------------------------------------------------+
| 入力受付状態: 受付中                                         |
+--------------------------------------------------------------+
| コート名        [ Aコート v ]                                |
| 対象試合        [ 第2試合 A3 vs A4 v ]                       |
| チームA         A3                                            |
| チームB         A4                                            |
| チームAスコア   [ 21 ]                                        |
| チームBスコア   [ 18 ]                                        |
|                                                            |
| [結果を送信]                                                |
+--------------------------------------------------------------+
| 入力完了: A3 vs A4 の結果を登録しました                      |
+--------------------------------------------------------------+
```

### 5.6 バリデーション仕様

| 条件 | エラー内容 |
| --- | --- |
| コート未選択 | コートを選択してください |
| 試合未選択 | 試合を選択してください |
| スコア未入力 | スコアを入力してください |
| スコアが数値でない | 数値で入力してください |
| 同点 | 勝敗が決まるスコアを入力してください |
| 受付停止中 | 現在は入力受付中ではありません |

### 5.7 操作仕様

| 操作 | 挙動 |
| --- | --- |
| コート選択 | 対象コートの未確定試合を読込 |
| 試合選択 | チーム名を自動表示 |
| 送信ボタン押下 | バリデーション後に結果を保存 |
| 保存完了 | 完了メッセージを表示し、一覧を更新 |

### 5.8 状態遷移

| 現在状態 | 操作 | 次状態 |
| --- | --- | --- |
| 初期 | コート選択 | 試合選択可能 |
| 試合選択可能 | 試合選択 | 入力可能 |
| 入力可能 | 正常送信 | 完了表示 |
| 入力可能 | 異常送信 | エラー表示 |

### 5.9 レスポンシブ方針

- モバイルではフォームを縦積みにする。
- 送信ボタンは画面幅いっぱいで押しやすいサイズにする。

## 6. P04 Admin ページ仕様

### 6.1 目的

大会運営者が試合登録、時刻設定、入力受付制御、スコア修正を行えるようにする。

### 6.2 想定利用者

- 大会運営者

### 6.3 主な表示ブロック

| ブロックID | ブロック名 | 内容 |
| --- | --- | --- |
| ADM-01 | ヘッダー | 大会名、共通ナビ |
| ADM-02 | 運営設定 | 入力受付状態、全体メッセージ |
| ADM-03 | 試合一覧 | 全コートの試合一覧 |
| ADM-04 | 編集フォーム | 新規登録、編集、スコア修正 |
| ADM-05 | コート所属管理 | コートごとの所属チーム管理 |

### 6.4 編集対象項目

| 項目 | 用途 |
| --- | --- |
| コート名 | 試合所属コートの設定 |
| 試合順 | 試合の並び順設定 |
| 開始予定時刻 | 固定タイムテーブル枠設定 |
| チームA名 | 対戦カード設定 |
| チームB名 | 対戦カード設定 |
| 試合状態 | 手動修正時の状態変更 |
| 入力受付状態 | 審判入力の開始・停止 |
| スコア | Admin による修正 |

### 6.5 ワイヤーフレーム

```text
+--------------------------------------------------------------+
| 大会名                                                       |
| [Index] [Tournament] [ScoreInput] [Admin]                  |
+--------------------------------------------------------------+
| 入力受付: [受付中にする] [停止する]                          |
| 運営メッセージ: [ 試合開始10分前に集合 ]                     |
+--------------------------------------------------------------+
| 試合一覧                                                     |
| No | コート | 時刻  | 対戦カード   | 状態   | 編集            |
| 01 | A      | 09:00 | A1 vs A2     | 終了   | [編集]          |
| 02 | A      | 09:20 | A3 vs A4     | 未開始 | [編集]          |
| 03 | B      | 09:00 | B1 vs B2     | 進行中 | [編集]          |
+--------------------------------------------------------------+
| 編集フォーム                                                 |
| コート名      [ Aコート v ]                                  |
| 試合順        [ 2 ]                                          |
| 開始予定時刻  [ 09:20 ]                                      |
| チームA       [ A3 ]                                         |
| チームB       [ A4 ]                                         |
| 状態          [ 未開始 v ]                                   |
| スコアA       [    ]  スコアB [    ]                         |
| [保存] [クリア]                                              |
+--------------------------------------------------------------+
| コート所属管理                                               |
| Aコート: A1, A2, A3, A4, A5, A6, A7                         |
| Bコート: B1, B2, B3, B4, B5, B6, B7                         |
+--------------------------------------------------------------+
```

### 6.6 操作仕様

| 操作 | 挙動 |
| --- | --- |
| 受付開始 | ScoreInput を有効化 |
| 受付停止 | ScoreInput を無効化 |
| 編集ボタン | 対象試合を編集フォームへ反映 |
| 保存ボタン | 試合情報または修正スコアを保存 |
| クリアボタン | フォーム入力を初期状態へ戻す |

### 6.7 制御ルール

- スコア修正は Admin 画面からのみ実施可能とする。
- 進行中または終了試合も Admin は編集可能とする。
- 受付停止中でも Admin による編集は可能とする。
- 同点スコアは登録不可とする。

### 6.8 レスポンシブ方針

- モバイルでは試合一覧と編集フォームを縦積みにする。
- PC では一覧と編集フォームを 2 カラム配置できるようにする。

## 7. データ反映ルール

### 7.1 ScoreInput からの反映

- 結果保存後、対象試合の状態を completed に更新する。
- 勝者を自動計算して保持する。
- 必要に応じて scoreLocked を true に変更する。

### 7.2 Admin からの反映

- 試合順、時刻、対戦カード変更後は Tournament と Index に反映する。
- 受付状態変更後は ScoreInput へ即時反映できる構造にする。

## 8. 今後の実装メモ

- 初期実装では仮データで画面再現を行う。
- API 連携時は表示仕様を変えず、データ取得元のみ差し替える。
- 40 チームのコート別配分は運営確定後に初期データへ反映する。
